# multi_agent_system
这个项目本质上是一个多 Agent 协同 + 自动模型分配的任务执行框架。 一、它解决的核心痛点：单一大模型“干所有事”效率低、成本高 传统做法是：一个GPT → 处理分析 + 写代码 + 决策 + 对话 问题：成本高（全部走高端模型）、不稳定（模型不擅长的任务也硬做）、不可控（没有分工） 这个项目的解决方式： 任务拆分 → 不同 Agent → 自动选最合适模型 这个项目提供： Agent注册机制（像插件） 任务路由（解耦） 模型选择器（策略层） 本项目支持： analysis_agent → coding_agent →（可继续扩展） 本质是：把AI从“单次回答”升级成“流水系统“
